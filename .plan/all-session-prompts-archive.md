## Bug Fixes (included in implementation)

### Bug 1: Password verification bypass (`service.rs:100`)
`verify_password` returns `Ok(false)` for wrong password, but `login_user` uses `?` which only propagates `Err`. Wrong passwords generate valid tokens.

**Fix**: Change `verify_password` to return `Err(AppError::Unauthorized("Invalid credentials"))` when password doesn't match, instead of `Ok(false)`. Update return type to `Result<(), AppError>`.

### Bug 2: Error swallowing in service methods (`service.rs:37-52`)
`let _ = with_transaction(...)` and `let _ = create_user(...)` discard all errors. Service methods always return `Ok(())` regardless of DB failures.

**Fix**: Remove all `let _ =` patterns. Propagate errors with `?`. Affected methods: `create_user`, `update_user`, `delete_user`. The `with_transaction` return type (`TxResult<T>`) needs to be mapped to `AppError` and propagated.

## Context

Identity's `main.rs` had to abandon `run_service()` and inline all bootstrap logic (tracing, DB, HTTP binding) just to add Redis and gRPC. Keep `run_service` as-is for HTTP-only services. Add a new `run_service_with_infra` that extends it with optional Redis init and optional gRPC server.

### API

```rust
// shared/src/server.rs

pub struct GrpcConfig {
    pub port_env_key: &'static str,
    pub default_port: u16,
}

pub async fn run_service_with_infra<F, G, Fut>(
    config: ServiceConfig,
    grpc: Option<(GrpcConfig, G)>,
    build_app: F,
) -> Result<(), Box<dyn Error>>
where
    F: FnOnce(PgPool, CommonAppState, Option<redis::aio::ConnectionManager>) -> Router,
    G: FnOnce(PgPool, SocketAddr) -> Fut,
    Fut: Future<Output = Result<(), tonic::transport::Error>> + Send,
```
